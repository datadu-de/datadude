What is he talking about?

As a data dude, dealing with messy data is daily life. And dealing means making it behave the way you need it to.

Luckily for me, what was a mess with custom scripts or, even worse, Excel Macros, has now become much more easy to deal with, since Power BI introduced PowerQuery, a.k.a. M, a.k.a. Get & Transform. And of course in Python there´s Pandas, which also allows for fast cleansing and reshaping while good old SSIS is declining fast. > Life is good then, right?

Sounds good, but not so fast - as data volume is getting bigger many work projects are now shifting towards MPP - embracing lake house architectures and Databricks which is Apache Spark under the hood. And while this is the latest and greatest tech, it lacks the convenience of some other tools I mentioned earlier.

Today, let's Unpivot Other Columns

One of the functions I frequently use in Pandas and in PowerQuery is Unpivot Other Columns. In Pandas it's called melt or wide to long. And while melt is part of koalas the pure spark-way of doing it feels clunky.

  "continent": ["Americas", "Americas", "Asia", "Europe"],
  "country": [ "USA", "Mexico", "Japan", "France"],
  "Population 2014": [1, 2, 3, 4],

from pyspark.sql import DataFrame
from typing import Iterable 

def unpivot_df(
        df: DataFrame, 
        id_cols: Iterable[str], pvt_cols: Iterable[str]=None,
        col_name: str="Column", val_name: str="Value") -> DataFrame:

  _df = df
  _cols = set(_df.columns)
  _id_cols = id_cols

  if pvt_cols is None:
    _pvt_cols = list(_cols.difference(id_cols))
  else:
    _pvt_cols = pvt_cols

  _stack_expr = ', '.join(["'{}', `{}`".format(_c, _c) for _c in _pvt_cols])
  _stack_expr = f"""stack({len(_pvt_cols)}, {_stack_expr}) as ({col_name}, {val_name})"""
  _unpvt_expr = _id_cols + [_stack_expr, ]
  _unpvt_df = _df.selectExpr(*_unpvt_expr)
  
  return _unpvt_df

spark_df_long = unpivot_df(spark_df, ["continent", "country"])
display(spark_df_long)
